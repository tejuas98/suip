SCRIPTIFY

REACT NATIVE (EXPO) CLOUD VISION API CUSTOM OCR MAPPING FIREBASE PYTHON

India has 22 official languages written in 13 different scripts. For a traveler arriving in an unfamiliar state, a street sign in Tamil or Devanagari is completely unreadable. SCRIPTIFY solves this by processing raw camera input through a cloud-based OCR inference layer, then instantly transliterating detected text into the user's native script — offline-capable, with no internet dependency for the transliteration step itself.

The pipeline: camera frame → image preprocessing (contrast enhancement, noise reduction) → YOLO-based text region detection → Cloud Vision OCR extraction → Aksharamukha transliteration engine → rendered output in Devanagari, Tamil, Gurmukhi, or Bengali.

I was the only developer on a 6-person team. I wrote all the code, managed the architecture decisions, debugged API failures at 3AM, and demonstrated a live working demo to the SIH evaluation panel within 36 hours of starting. The app functioned. That's the metric that mattered.

SYSTEM ARCHITECTURE

The original approach used only Tesseract OCR offline — acceptable accuracy for clean signage, unacceptable for low-contrast or damaged signs. Switched to Cloud Vision API mid-sprint after testing showed 40% improvement on real street sign images. The offline transliteration step (Aksharamukha) stayed on-device so the app works without connectivity after the first API call.

PLAN KARO

PRODUCT ARCHITECTURE UI/UX DESIGN LEAN CANVAS MARKET STRATEGY

In Tier-2 Indian cities, small event organizers — college fests, family functions, local cultural programs — have no structured way to discover and book local vendors: decorators, caterers, sound systems, photographers. They rely on WhatsApp referrals and cold calls. Vendors, on their side, have no consistent discovery channel.

PLAN KARO is a B2B digital marketplace that bridges this gap. The business model is built around a 12% vendor commission on confirmed bookings — a number derived from actual market research on existing event agency margins in Nashik, Aurangabad, and Bhusawal during the bootcamp.

The platform architecture: vendor onboarding → service catalog → organizer discovery → quote request → booking confirmation → commission settlement. The UX is deliberately lean — we found that Tier-2 event organizers distrust complex platforms. Every screen has one primary action.

DATE: 2025–2026 | TYPE: Data Pipeline + Report Engine | STACK: Vanilla JS, HTML5/CSS3, Node.js, Tailwind CSS | CONSTRAINT: Zero external chart libraries

IKS WEATHER REPORT BUILDER

HTML5 / CSS3 VANILLA JS NODE.JS UI DESIGN

For the IKS Bucket Assignment, I needed a way to take raw hourly weather data — temperature, humidity, wind speed, pressure — and produce a print-ready analytical report instantly, without setting up a backend or depending on a paid charting service.

The solution: a browser-based data pipeline. User inputs hourly values manually or pastes from a spreadsheet. User selects color themes for each graph series. The system generates a full-page PDF-ready report — axes, legends, annotations, title block, and all — using canvas-based rendering written from scratch in Vanilla JS.

No Chart.js. No D3. Every graph element is drawn directly on an HTML canvas using calculated coordinates. This constraint made the project significantly harder and significantly more instructive.
